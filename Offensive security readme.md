## What is Offensive Security?

Offensive Security means **legally and ethically attacking systems** to find weaknesses before real attackers do.

People in this field think like criminals — but work for the good side. They simulate real-world attacks, find vulnerabilities, write detailed reports, and help organizations fix their security gaps before someone malicious finds them first.

> **Core idea:** Find it before the bad guys do.

This is not just "hacking." It is a discipline — with methodology, ethics, legal boundaries, documentation, and professional standards.

---

## Is This Legal?

**Yes — when done with written permission.**

Offensive security professionals always work under:
- A signed contract or scope of work
- A defined Rules of Engagement (ROE) document
- Legal authorization from the target organization

Without permission = illegal. With permission = professional.

## The 7-Stage Learning Roadmap

Every offensive security professional — no matter the role — goes through these stages. Some stop at Stage 3. The elite reach Stage 7.

| Stage | What You Learn | What You Build |
|-------|---------------|----------------|
| **Stage 1** — Foundations | Networking, OS, Linux, scripting, basic pentesting concepts | 1 recon report + 1 vulnerability writeup |
| **Stage 2** — App & Network Testing | OWASP Top 10, exploitation basics, web + infra testing | 1 web app report + 1 infrastructure report |
| **Stage 3** — Post-Exploitation | Active Directory, privilege escalation, lateral movement, pivoting | 1 full AD lab compromise report |
| **Stage 4** — Exploit Dev & Tooling | Reverse engineering, assembly, payload writing, custom tools | 1 custom exploit or payload PoC |
| **Stage 5** — Adversary Simulation | Campaign planning, stealth ops, detection evasion, C2 usage | 1 complete red team playbook |
| **Stage 6** — Specialization | Cloud attacks / IoT / RF / Firmware / Hardware | 1 specialized research report |
| **Stage 7** — Leadership & Research | Run red teams, publish research, speak at conferences | 1 public whitepaper or conference talk |

## How to Enter Offensive Security

You do not need a degree. Most professionals start from one of these paths:

| Entry Path | What It Means |
|------------|---------------|
| **Self-learning** | TryHackMe, HackTheBox, PortSwigger — free labs online |
| **CTF competitions** | Capture The Flag — legal hacking challenges |
| **Bug bounty** | Find real bugs in real apps, earn money, build reputation |
| **Certifications** | eJPT → OSCP — structured learning with credentials |
| **Home lab** | Build your own practice environment using virtual machines |
| **Internship / Junior role** | Get hired entry-level, learn on the job |

## Skill Foundation (Required Before Any Role)

These are not optional. Every offensive security role requires these basics:

<details>
<summary><b>Networking Basics</b></summary>

<br>

- How TCP/IP works — packets, handshakes, routing
- DNS, HTTP/S, FTP, SSH, SMB — what they are and how they work
- Ports and protocols — what runs where
- Firewalls, proxies, VPNs — how they block and how to work around them
- Tools: Wireshark, Nmap, Netcat

</details>

<details>
<summary><b>Operating Systems</b></summary>

<br>

- **Linux** — command line, file system, permissions, processes, bash scripting
- **Windows** — Active Directory basics, registry, services, PowerShell
- Understand how both systems store credentials, manage users, and handle processes

</details>

<details>
<summary><b>Web Technologies</b></summary>

<br>

- How HTTP requests and responses work
- Cookies, sessions, authentication flows
- HTML, JavaScript — enough to understand what a web app is doing
- Databases — basic SQL, how queries work

</details>

<details>
<summary><b>Security Concepts</b></summary>

<br>

- CIA Triad — Confidentiality, Integrity, Availability
- OWASP Top 10 — most common web vulnerabilities
- CVE system — how vulnerabilities are tracked and rated
- Basic understanding of how exploits work

</details>

<details>
<summary><b>Scripting</b></summary>

<br>

- **Python** — write simple scripts, automate tasks, modify existing tools
- **Bash** — automate Linux tasks, write recon scripts
- **PowerShell** — Windows automation, AD interaction

</details>

## Portfolio — What You Must Build

Before applying for any offensive security role, build these:

| # | Deliverable | What It Shows |
|---|-------------|---------------|
| 1 | 3 complete pentest reports (Web + Infra + AD) | You can find, document, and communicate findings |
| 2 | 1 exploit PoC or custom script | You understand how attacks work technically |
| 3 | 1 red team campaign summary with detection map | You can plan and execute structured operations |
| 4 | 1 cloud or IoT assessment | You have modern attack surface knowledge |
| 5 | Public GitHub / blog with sanitized writeups | You can communicate and teach |

## Common Traps — Brutal Truth

| Trap | Reality |
|------|---------|
| Jumping into Metasploit first | You will not understand what is happening under the hood |
| Ignoring defense and detection | Real attackers study what gets them caught |
| Buying tools before building skills | Tools amplify knowledge — they do not replace it |
| Collecting certificates without labs | Certificates without hands-on skill mean nothing |
| Skipping report writing | If you cannot document it, you cannot get paid for it |
| "I am a hacker so I do not need soft skills" | Every client engagement needs communication |
